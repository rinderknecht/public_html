Converting from binary to hexadecimal

Question. The purpose of this exercise is to write a C program which reads on the command-line a string representing a positive integer in binary form and outputs its corresponding hexadecimal representation.

Hints and constraints.
  • The input must be checked in the main function, before the computation takes place, in the from2to16 function. In particular, be aware of invalid characters. (The empty string is not possible since the input is given on the command line.) Also, in order to find the number of significant bits, the leftmost zeros must be discarded.
  • The signature of from2to16 must be char* from2to16(const char bin[], unsigned int b_len), where bin is the binary string and b_len is the length of bin. The length is computed in main, when checking the input string, and it is useful in from2to16 to know the number of hexadecimal digits, thus how to allocate the output string.
  • Write a function whose signature is char mk_digit(const char bin[], unsigned int len), where bin is a binary string and len is the number of leftmost bits to be converted to an hexadecimal digit. Clearly, len ranges from 0 to 4, both included. For example, if bin is the string "11010011111", then mk_digit(bin,3) == '6', which is the decimal form of 110 (the three leftmost bits). Also, mk_digit(bin+6,4) == 'F'.
  • The difficulty lies in handling the case where the number of bits are not a multiple of four. The remaining leftmost bits need special attention. In function main, the difficulty is to ignore the leftmost zeros before passing the binary string to from2to16.
