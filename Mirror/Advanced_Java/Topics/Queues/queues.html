Queues

Queues are, with stacks, the other linear data structure commonly used by programmers. The difference between a queue and a stack is that keys are pushed on a queue at the opposite end where they are popped. Pushing in this case is called enqueueing and popping is called dequeueing. Consequently, as far as data fields are concerned, the class modelling queues contains two references: one on the last enqueued object (on the rear), and one on the first object to be dequeued (on the front). (Note that these references must either be both non-null or null.) Having two references instead of one, as with stacks, enables constant time enqueueing and dequeueing.

Persistent Queues (functional style)

We follow the same design pattern as with the persistent stacks, that is, we have an abstract class Queue from which two final classes are derived: one for the empty queues (EQueue) and one for the non-empty queues (NQueue). Note that we implement both enqueue and dequeue, and the latter being partial, throws the exception QEmpty.

The technique to implement non-empty queues consists in have a pair of stacks, one to dequeue from, called the front queue, and one to enqueue to, called the rear stack. A description of this implementation and its amortised cost can be found in my article A Didactic Analysis of Functional Queues. The Java classes are as follows.
