Arborescence de processus pipés

cat | tail | wc

1. Cette commande affiche sur sa sortie standard le nombre de lignes, de mots et de caractères des au plus 10 dernières lignes qu'elle a lues sur son entrée standard.

Si le nombre de lignes entrées est <10, le nombre de lignes calculé sera le nombre de lignes entrées. Si le nombre de lignes entrées est >=10, le nombre de lignes calculé sera 10

On rappelle que par defaut, l'entrée standard, la sortie standard et la sortie erreur standard sont associées au terminal, i.e. par défaut, l'entrée standard correspond à ce qui est entré au clavier, la sortie standard et la sortie erreur standard s'affichent sur un terminal (un xterm par exemple).

Dépendant du système, une commande alternative est : ps -l -u <username>
-l : option longue avec PID et PPID
-u : option avec valeur <username> : les processus de l'utilisateur.

2. la commande ps -ef permet de lister, notamment avec les informations PID (numero du processus) et PPID (numero du processus pere) tous les processus du systeme.

La commande ps -ef | grep -i <username> permet de lister, avec ces memes informations, tous les processus apartenant a l'utilisateur <username>

ex: ps -ef | grep -i thummel permet de lister tous les processus apartenant a l' utilisateur thummel.

Cette commande permet de voir qu'ici les commandes cat, tail et wc sont lancées en parallèle et les processus correspondant sont tous des fils du shell (bash) les ayant lancés (i.e. ayant lancé la commande cat | tail | wc).

En effet, ces 3 processus figurent bien simultanément tous trois dans la liste des processus (avec bien sur des PID différents) avec un PPID identique égal au PID du shell les ayant lancés.

3. les commandes cat, tail et wc sont toutes les trois implementées comme des "filtres" : elles lisent sur leurs entrées standard et écrivent toutes les 3 sur leur sorties standard

Lorsque l'on lance cat | tail | wc :

  • 2 tubes (pipes) et trois processus sont crées par le shell interprétant cette commande
  • la sortie standard de cat est connectée a "l'entrée" du premier tube (cat écrit dans le premier tube)
  • l'entrée standard de tail est connectée a la "sortie" du premier tube (tail lit dans le premier tube)
  • la sortie standard de tail est connectée a "l'entrée" du second tube (tail écrit dans le second tube)
  • l'entrée standard de wc est connectée dans la "sortie" du second tube (wc lit dans le second tube)

a) on tue (kill -15 <pid> ou kill -s TERM <pid> ou encore kill -s sigterm <pid>) wc

tail tente alors d'écrire dans un tube qui n'a plus de lecteur. Le processus correspondant a tail recoit le signal SIGPIPE - situation de "broken pipe", (cf man 7 signal :

SIGPIPE 13 Term Broken pipe: write to pipe with no readers ).

Sur le terminal, s'affiche le message "Terminated" (ou "Killed" si on a fait un kill -9 au lieu d'un kill -15) correspondant a l'arret de wc

La commande tail se termine en appelant la fonction exit

b) on tue tail (après avoir rentré des caractères et avant d'avoir pressé Ctl-D)

Il n'y a plus d'écrivain sur le tube dans lequel lit wc. wc recoit à travers le tube un caractere EOF (fin de fichier) signe qu'il n'y a plus rien à lire et donc renvoie sur la sortie standard le calcul (nb de lignes,mots,octets) portant sur ce qu'il a lu, c'est à dire dans ce cas rien (car head attendait de lire un caractere EOF (celui qu'elle aurait recu en temps normal suite a l'appui de Ctl-D) pour ecrire dans le tube).

wc affiche donc 0 0 0 sur la sortie standard

c) on tue cat

On a une situation similaire a b).

Le tube dans lequel lit tail n'a plus d'ecrivain. tail lit un caractère EOF, retourne (aucune ligne lue), wc retourne alors et écrit 0 0 0 sur la sortie standard.

Script datef.sh

Le script datef.sh

L'idee generale est de :

  • a) récuperer les différents champs (jour, mois) affiches par la commande unix "date"
  • b) utiliser la structure de contrôle "case ... esac" pour regénerer, en fonction de la valeur des champs a traduire (ceux qui contiennent une chaine de caractère en anglais) et en prévoyant tous les cas possibles, des chaines de caractères "francaises"

Pour a), on utilise :

  • les backquotes (`) qui ont pour effet de faire executer par le shell la commande date et de remplacer `date` par le resultat de cette execution
  • la commande (interne au shell) set permettant de réassigner les parametres positionnels $1, $2, $3, ... correspondant normalement a ce que l'on passe en paramètre a la commande (i.e. ici datef.sh).
  • le fait que les champs affichés par la commande "date" sont sépares par des caracteres "espace" qui est egalement le séparateur du shell (caractère qui sépare les commandes et les arguments), assurant ainsi le bon fonctionnement de la commande "set"

Pour exécuter le script, il faut

  • rendre le fichier exécutable (chmod 755 datef.sh)
  • lancer la commande ./datef.sh si l'on est dans le repertoire le contenant
